# 动效设计原则

## 1. 目的导向

每个动效必须回答:**它解决了什么问题?**

| 用途 | 动效目的 |
|---|---|
| 反馈 | 让用户知道操作生效(按钮按下) |
| 状态切换 | 引导用户视线(Tab 切换) |
| 空间关系 | 表达层级(模态从底部升起) |
| 加载占位 | 让等待变得可承受(Skeleton shimmer) |
| 情感表达 | 营造氛围(大促礼花 / 加购成功 bounce) |

**反例**:动效**只是好看**,无具体功能 → 删掉。

---

## 2. 克制

京东不是动效驱动的产品。**主流程动效尽量少 / 短**:
- PDP / 购物车 / 结算 / 支付:动效 ≤ 200ms,无装饰性动效
- 大促首屏 / 节庆专题:可以用更丰富动效
- 工具型页面(订单 / 设置):极简,只保留状态切换

---

## 3. 物理感

动效要符合物理直觉:
- 卡片飞动画沿真实物理路径(贝塞尔曲线,非直线)
- 弹性反馈用 spring 缓动(自然回弹)
- 重物慢启动 + 轻物快启动

**京东不用拟物化动效**(如 3D 翻转 / 真实纸张),保持扁平视觉一致。

---

## 4. 时序一致

同一交互序列动效时序对齐:
- 模态进入 200-300ms,蒙层与内容同步淡入
- Tab 切换 200ms,不允许"内容到了才出现"
- 加购飞动画总时长 500ms,不允许"商品图飞了 800ms 才落"

---

## 5. 性能优先

动效不能影响交互:
- 60 FPS 是底线,不允许掉帧动效上线
- 列表滚动中不触发额外动效
- 大量元素同时动画使用 GPU 优化属性(transform / opacity)

---

## 6. 减少动效响应

所有非必需动效必须能在系统"减少动效"开启时退化:
| 默认 | 减少动效后 |
|---|---|
| 模态滑入 | 淡入 |
| 加购飞动画 | 直接 Toast |
| 大促礼花 | 不显示 |
| 转场动画 | 直接切换 |

---

## 7. 引用此原则的场景

- 评审任何带动效的方案 → 走 5 条原则 checklist
- 大促设计 → 在"克制"和"情感表达"之间找平衡
- 性能优化 → 物理感 vs 60 FPS 的取舍

---

## 8. 原则间张力

**克制 vs 情感表达**:大促首屏需要氛围,主流程需要克制 → 看场景
**物理感 vs 性能**:贝塞尔曲线物理感强但开销大 → 移动端用预设的 spring token
