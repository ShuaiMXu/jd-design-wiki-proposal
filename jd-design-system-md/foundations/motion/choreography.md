# 动效编排 · Choreography

> 多个元素的动效如何协调。**编排错 = 视觉混乱 = 用户分不清主次**。

---

## 1. 阶梯入场(Staggered)

列表 / 卡片入场时,逐个错开:

```
卡 1: 0ms 入场
卡 2: 50ms 入场
卡 3: 100ms 入场
卡 4: 150ms 入场
...
```

- 间隔:50-80ms
- 每张卡时长:`fast` (200ms)
- 总时长:不超过 800ms(否则等待感强)

---

## 2. 主从关系

主元素先动,从元素跟随:

| 主元素 | 从元素 | 间隔 |
|---|---|---|
| 模态进入 | 蒙层淡入 | 同步 |
| Tab 切换 | 内容渐变 | 同步 |
| Sheet 滑入 | Sheet 内 CTA 渐入 | Sheet 完成后 100ms |
| 提交订单成功 | 跳转动画 | Toast 完成后 |

---

## 3. 转场关系

页面转场时元素如何对接(共享元素动画):

| 场景 | 共享元素 |
|---|---|
| 列表 → PDP | 商品图(放大 + 移动到顶部) |
| 头像 → 个人主页 | 头像(放大)|
| 卡片 → 详情 | 卡片背景(扩展到全屏)|

**实现**:iOS Hero Animation / Android Shared Element Transition。

---

## 4. 退出反向

进入时阶梯入场 → 退出时反向阶梯出场。

```
进入:卡 1 → 卡 2 → 卡 3 → 卡 4
退出:卡 4 → 卡 3 → 卡 2 → 卡 1(逆序)
```

视觉上闭环,用户感觉"整理整齐"再消失。

---

## 5. 同时 vs 错开

| 场景 | 选择 |
|---|---|
| 列表初次加载 | 错开(阶梯) |
| 列表更新(刷新)| 同时(整体淡入) |
| 多个 Toast | 不同时(队列) |
| 多个角标变化 | 同时(同步) |

---

## 6. 反例

| ❌ 反面 | 解释 |
|---|---|
| 阶梯入场总时长 > 800ms | 等待感强 |
| 同时入场 10+ 元素(无错开) | 视觉冲击大 |
| 主从顺序反了(从元素先动) | 视觉混乱 |
| 进入退出方向不一致 | 节奏感差 |
